# 04-HTTP-FLV协议

该协议基于HTTP模式,开发HTTP-FLV协议的主要目的是为了解决两个问题
HLS协议的延迟率问题, 
RTMP的语法API难用的问题
同时,HTTP-FLV协议也是实时的,
格式是flv
主要是建立了长链接

## 优点
可以在一定程度上避免防火墙的干扰,(有的机房只允许80端口通过)
可以很好的兼容HTTP302跳转,做到灵活调度,比如,某个CDN节点挂了,那么可以直接很方便的302进行临时跳转到有效资源的CDN上面
可以直接使用HTTPS做加密通道处理
很好的支持移动端(Android, IOS)
